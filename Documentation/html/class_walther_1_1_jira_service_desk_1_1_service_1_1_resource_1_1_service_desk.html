createRequestType ( int  $serviceDeskId,
string  $issueTypeId = '',
string  $name = '',
string  $description = '',
string  $helpText = '' 
)

Create request type

This method enables a customer request type to be added to a service desk based on an issue type. Note that not all customer request type fields can be specified in the request and these fields are given the following default values:

  • Request type icon is given the question mark icon.
  • Request type groups is left empty, which means this customer request type will not be visible on the customer portal.
  • Request type status mapping is left empty, so the request type has no custom status mapping but inherits the status map from the issue type upon which it is based.
  • Request type field mapping is set to show the required fields as specified by the issue type used to create the customer request type. These fields can be updated by a service desk administrator using the Request types option in Project settings. Request Types are created in next-gen projects by creating Issue Types. Please use the Jira Cloud Platform Create issue type endpoint instead.
See also
https://developer.atlassian.com/cloud/jira/service-desk/rest/#api-rest-servicedeskapi-servicedesk-serviceDeskId-requesttype-post
